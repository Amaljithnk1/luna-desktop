# Competitive Defense Notes

This document is for internal/demo preparation.

## If another team has voice

Luna has Voice mode with push-to-talk where available, transcript fallback everywhere, optional spoken response, and feminine voice preference.

## If another team has an orb/globe

Luna has an always-on-top orb companion window, in-app orb fallback, global shortcut and command palette.

## If another team has PDF export

Luna exports PDF, DOCX, PPTX, HTML, Markdown, CSV, JSON, ICS and ZIP.

## If another team has screen understanding

Luna has Luna Lens: bounded active-window/running-app context, manual screenshot/image import, local OCR and context explanation.

## If another team has file automation

Luna has preview, approval, manifest logging, full mission undo and privacy trace.

## If another team has memory

Luna has reviewable memory, embedding/keyword retrieval, adaptive context and memory toggle.

## If another team has knowledge base

Luna has Knowledge Vault with PDF/DOCX/TXT/MD/CSV/JSON import, embeddings if available, keyword fallback and evidence cards.

## If another team has job/resume tools

Luna has Job Application Mission using imported attachments when available and generating PDF/DOCX/ZIP outputs.

## If another team has meeting/invoice/study/code tools

Luna has Mission Hub and Luna Skill Creator covering Meeting Notes, Invoice Extractor, Study Pack and Codebase Explainer.

## If another team claims privacy-first

Luna has external request counter, audit log, Trust Center, SQLite status, trust export, delete/reset and per-workflow privacy traces.

## If another team has more features

Luna's defense is not only feature count. Luna has feature-generating systems:

- Command Router
- Luna Skill Creator
- Mission Hub
- Unified Attachments
- Artifact Studio
- AI Demo Presenter

## If demo environment is weak

Luna has fallback mode, seeded demo data, reset state, and Judge Showcase / AI Presenter.
